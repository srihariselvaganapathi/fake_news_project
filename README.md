# Fake News Detector API

A compact machine-learning project that trains a text classifier to label news articles as **REAL** or **FAKE**, then exposes the trained model through a FastAPI prediction endpoint.

The repository is intentionally small: it contains the text-cleaning pipeline, model-training script, serialized model artifacts, and API service code needed to run local predictions.

## What This Project Does

- Cleans raw news text by lowercasing, removing URLs/HTML-like tags, removing non-letter characters, and normalizing whitespace.
- Trains a TF-IDF + Passive Aggressive Classifier pipeline on separate real and fake news CSV files.
- Saves the fitted classifier and vectorizer as `joblib` artifacts.
- Serves predictions through a FastAPI application with automatic Swagger documentation.

## How It Works

### 1. Text preprocessing

Input text is normalized before training and prediction. The cleaning function:

1. Converts input to lowercase.
2. Removes URL-like content.
3. Removes HTML-like tags.
4. Removes most non-letter characters.
5. Collapses repeated whitespace.

This same function is used by both the training script and the API, which keeps training-time and inference-time preprocessing aligned.

### 2. Model training

The training script expects two CSV files in `data/`:

- `data/true.csv` for real news rows.
- `data/fake.csv` for fake news rows.

Each CSV must include a `text` column. During training, the script labels `true.csv` rows as `REAL`, labels `fake.csv` rows as `FAKE`, shuffles the combined dataset, splits it into train/test sets, vectorizes text with TF-IDF, trains a `PassiveAggressiveClassifier`, prints accuracy, and writes model artifacts to `models/`.

### 3. API inference

The FastAPI application loads these artifacts at startup:

- `models/fake_news_model.pkl`
- `models/tfidf_vectorizer.pkl`

The `/predict` endpoint accepts JSON with a `text` field, cleans the text, vectorizes it, and returns the predicted label plus the processed text.

## Requirements

- Python 3.10+ recommended.
- `pip` and a virtual environment tool.
- Training data in `data/` if you want to retrain the model.

Install dependencies from the existing dependency file:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requiremts.txt
```

> **Note:** The dependency file is named `requiremts.txt` in this repository. If you prefer the conventional name, rename it to `requirements.txt` and update commands accordingly.

## Dataset Setup

Create a `data/` directory and place the expected CSV files inside it:

```text
data/
├── true.csv
└── fake.csv
```

Minimum required schema:

| Column | Required | Description |
| --- | --- | --- |
| `text` | Yes | Full article body or news text used for training. |

The `data/` directory is ignored by git, so datasets can be kept local and excluded from commits.

## Train the Model

Run the training script from the repository root:

```bash
python -m src.train_model
```

Expected behavior:

1. Reads `data/true.csv` and `data/fake.csv`.
2. Cleans all text.
3. Trains the TF-IDF vectorizer and classifier.
4. Prints test accuracy.
5. Saves updated artifacts in `models/`.

If the data files are missing, the script prints an error and exits without training.

## Run the API

Start the FastAPI server from the repository root:

```bash
uvicorn app.main:app --reload
```

Then open:

- Swagger UI: <http://127.0.0.1:8000/docs>
- Root URL: <http://127.0.0.1:8000/> redirects to `/docs`

## Prediction Endpoint

### `POST /predict`

Request body:

```json
{
  "text": "Breaking news article text goes here."
}
```

Example request:

```bash
curl -X POST "http://127.0.0.1:8000/predict" \
  -H "Content-Type: application/json" \
  -d '{"text":"Government officials announced a new policy today after months of debate."}'
```

Example response shape:

```json
{
  "prediction": "REAL",
  "processed_text": "government officials announced a new policy today after months of debate"
}
```

The returned `prediction` value is generated by the saved classifier and is usually either `REAL` or `FAKE`.

## Development Notes

- Run commands from the repository root so relative paths like `data/` and `models/` resolve correctly.
- Retraining overwrites the existing files in `models/`.
- The current classifier does not return probability scores; the API returns only the predicted class and cleaned text.
- The cleaning regex currently keeps only a subset of alphabetic characters and whitespace, so review `src/preprocess.py` before using this project for production or multilingual text.
- The serialized artifacts in `models/` should be regenerated whenever the preprocessing, vectorizer, model hyperparameters, or training data change.

## Limitations and Responsible Use

This project is a learning-oriented fake-news classifier, not a production fact-checking system. Model predictions can be wrong, especially for:

- Out-of-distribution topics or writing styles.
- Satire, parody, opinion pieces, or mixed-factuality articles.
- New events that differ from the training data.
- Short snippets without enough context.

Use predictions as a signal for experimentation or triage, not as the only source of truth. For real-world use, add dataset documentation, stronger evaluation, calibration, explainability, monitoring, and human review.

## Troubleshooting

| Problem | Likely cause | Fix |
| --- | --- | --- |
| `ModuleNotFoundError` | Dependencies are not installed. | Activate your virtual environment and run `pip install -r requiremts.txt`. |
| `Error:CSV file not found in 'data/' folder` | Training CSVs are missing. | Add `data/true.csv` and `data/fake.csv`. |
| API returns `Model not loaded.` | Model artifacts are missing or failed to load. | Run `python -m src.train_model` and restart the API. |
| `uvicorn` command not found | Uvicorn is not installed in the active environment. | Install dependencies in the active virtual environment. |
